The Unit Test Case Generator

Did you ever have a bunch of unit tests that have exactly the same logic but different inputs and outputs? Do you hate having to write and re-write exactly the same code over and over again? Well, you're in luck!
The Unit Test Case Generator takes two blocks of text: the code for your logic, or whatever you want to duplicate with modifiable portions. For any substitutions you want to make, just write a identifier without spaces between two dollar signs, like $this$. Then, in the second box, write a line containing each substitution label, followed by a space, followed by the thing you want to replace it with, then another identifier, as many as necessary.

Test Cases:

Tips

  • Substitutions must look like this: $identifier$. They must not have spaces, line breaks, or any other whitespace.
  • Each line in the Test Cases textbox is one test case. Each substitution present in the Test Method textbox must appear once (although not necessarily in order), separated by spaces. See the Example below.
  • Here's a list of escapes that work in both the Test Method textbox and the Test Cases textbox:
    • %dollar becomes $
    • %percent becomes %

Example

Test Method
[TestCase]
public void IsLeapYear_$Name$()
{
int year = $Year$
bool value = IsLeapYear(year);
Assert.Equals(value, $Expected$);
}
Test Cases
$Name$ BeforeLeapYears $Year$ 1750 $Expected$ false
$Name$ Mod4 $Year$ 2012 $Expected$ true
$Name$ NotMod100 $Year$ 1900 $Expected$ false
$Name$ ButMod400 $Year$ 2000 $Expected$ true
